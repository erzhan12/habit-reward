# Feature 0035: Claimed Rewards Menu Button

Add a "Claimed Rewards" button to the Rewards submenu that shows a simple list of one-time (`is_recurring=False`) claimed rewards. Recurring rewards reset after claiming and reappear in the normal My Rewards list, so only non-recurring claimed rewards need this separate view.

## Relevant Files

### Repository Layer
- **`src/core/repositories.py`** вАФ `RewardProgressRepository`
  - Add `get_claimed_non_recurring_by_user(user_id)` method
  - Query: `RewardProgress.objects.filter(user_id=user_pk, claimed=True, reward__is_recurring=False).select_related("reward", "user")`
  - Attach cached `pieces_required` via `_attach_cached_pieces_required()`

### Service Layer
- **`src/services/reward_service.py`** вАФ `RewardService`
  - Add `get_claimed_one_time_rewards(user_id)` method
  - Calls `self.progress_repo.get_claimed_non_recurring_by_user(user_id)`
  - Coerces results via `_coerce_progress()`
  - Returns list sorted by reward name (alphabetical)

### Bot Messages
- **`src/bot/messages.py`** вАФ `Messages` class + `_TRANSLATIONS`
  - Add `BUTTON_CLAIMED_REWARDS` вАФ Button text for the Rewards submenu (en: `"рЯПЖ Claimed Rewards"`, ru: `"рЯПЖ –Я–Њ–ї—Г—З–µ–љ–љ—Л–µ –љ–∞–≥—А–∞–і—Л"`, kk: `"рЯПЖ –Р–ї—Л–љ“У–∞–љ –Љ–∞—А–∞–њ–∞—В—В–∞—А"`)
  - Add `HEADER_CLAIMED_REWARDS` вАФ Header for the claimed rewards list (en: `"рЯПЖ <b>Claimed Rewards</b>"`, with translations)
  - Add `INFO_NO_CLAIMED_REWARDS` вАФ Empty state message (en: `"No claimed rewards yet. Claim one-time rewards to see them here!"`, with translations)

### Bot Keyboards
- **`src/bot/keyboards.py`**
  - Add button to `build_rewards_menu_keyboard()`: new row with `BUTTON_CLAIMED_REWARDS` and callback_data `"menu_rewards_claimed"`
  - Place it after the "Claim Reward" button and before the "Back" button

### Bot Formatters
- **`src/bot/formatters.py`**
  - Add `format_claimed_rewards_message(progress_list, rewards_dict, language)` function
  - Format each claimed reward as: `"рЯПЖ <b>{reward_name}</b> вАФ {pieces_required} pieces"`
  - Include the claimed rewards header from messages

### Bot Handlers
- **`src/bot/handlers/reward_handlers.py`**
  - Add `claimed_rewards_command(update, context)` handler вАФ for `/claimed_rewards` command
  - Add `menu_claimed_rewards_callback(update, context)` handler вАФ for menu button callback
  - Both handlers:
    1. Validate user exists and is active
    2. Call `reward_service.get_claimed_one_time_rewards(user.id)`
    3. If empty, show `INFO_NO_CLAIMED_REWARDS` with Back button
    4. If non-empty, format with `format_claimed_rewards_message()` and show with Back button

### Menu Handler Registration
- **`src/bot/handlers/menu_handler.py`**
  - Add `menu_rewards_claimed` to `bridge_command_callback` regex pattern
  - Add mapping entry: `'menu_rewards_claimed': claimed_rewards_command`
  - Import `claimed_rewards_command` from `reward_handlers`

## Algorithm

1. User taps "Claimed Rewards" button in Rewards submenu
2. `bridge_command_callback` routes `menu_rewards_claimed` to `claimed_rewards_command`
3. Handler validates user, calls `reward_service.get_claimed_one_time_rewards(user.id)`
4. Repository queries `RewardProgress` where `claimed=True` AND `reward__is_recurring=False` for the user
5. Service coerces results, sorts alphabetically by reward name
6. Handler formats the list (or shows empty state) and displays with a Back button
