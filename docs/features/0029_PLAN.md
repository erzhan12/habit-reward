# Feature 0029: Fix "Reward progress not found" Error on Back Button

## Description

When navigating **Rewards → Claim Reward → Back**, the user sees "Reward progress not found" error instead of returning to the Rewards menu.

## Root Cause

Pattern matching order bug in the `claim_reward_conversation` ConversationHandler. The regex `^claim_reward_` matches callback data `"claim_reward_back"` before the more specific pattern `^claim_reward_back$` can be evaluated.

**Flow when bug occurs:**
1. User clicks Back button (callback_data = `"claim_reward_back"`)
2. Pattern `^claim_reward_` matches first (greedily captures "claim_reward_back")
3. `claim_reward_callback()` is invoked instead of `claim_back_callback()`
4. Handler extracts reward_id: `"claim_reward_back".replace("claim_reward_", "") = "back"`
5. Tries to mark reward "back" as claimed → "Reward progress not found" error

## Files to Modify

### `src/bot/handlers/reward_handlers.py`

**Location:** Lines 2193-2197

**Current Code:**
```python
states={
    AWAITING_REWARD_SELECTION: [
        CallbackQueryHandler(claim_reward_callback, pattern="^claim_reward_"),
        CallbackQueryHandler(claim_back_callback, pattern="^claim_reward_back$")
    ]
}
```

**Required Change:** Swap handler order so more specific pattern is evaluated first:
```
1. claim_back_callback with pattern ^claim_reward_back$ (FIRST - exact match)
2. claim_reward_callback with pattern ^claim_reward_ (SECOND - prefix match)
```

## Algorithm

python-telegram-bot ConversationHandler evaluates handlers in list order. First matching pattern wins. When handlers share prefix patterns, the more specific pattern must come first.
