# Feature 0044: Dim Completed Today Streak Indicator

## Description

In the web interface, on the Today section, completed habits already show the habit name as gray and struck through. The current streak label and fire emoji still stay visually bright. Update Today habit cards so that when a habit is marked done, only the streak block (`🔥` and `N-day streak`) becomes a more muted gray. This should make habits that are not yet done stand out more clearly.

Scope from the request:

- Apply this only to the Today page.
- Change only the streak block for now, not the weight/reward-chance badge.
- The completed streak block should turn gray; keep the existing streak animation class applied.
- Use a gray that is more muted than the current completed habit name color.

## Files to Modify

### `frontend/src/components/HabitCardContent.vue`

This component renders the Today habit card content used by `frontend/src/components/HabitCard.vue`.

Relevant current behavior:

- The habit name already uses `habit.completedToday ? 'line-through text-text-secondary' : 'text-text-primary'`.
- The streak block is rendered when `habit.streak > 0`.
- The fire emoji and streak text currently both use `text-streak-fire` and `streakClass`, so completed habits keep the bright fire color:
  - `🔥`
  - `{{ habit.streak }}-day streak`

Required changes:

- Keep the `v-if="habit.streak > 0"` behavior unchanged.
- Add conditional classes for the streak emoji and streak text based on `habit.completedToday`.
- Use a mutually exclusive color class binding, e.g. completed uses `text-text-muted` and incomplete uses `text-streak-fire`; do not leave the static `text-streak-fire` class on completed streak elements.
- Move `text-streak-fire` out of the static `class` attribute and into the `:class` binding on both streak elements.
  - Static classes should keep only non-color utilities such as `text-sm`, `text-xs`, and `font-medium`.
  - The bound class array should include both the conditional color class and `streakClass`, for example:
    - `:class="[habit.completedToday ? 'text-text-muted' : 'text-streak-fire', streakClass]"`
- For incomplete habits, keep the current visual behavior:
  - `text-streak-fire`
  - existing `streakClass`
- For completed habits, replace the bright streak color with `text-text-muted`.
  - Keep applying the existing `streakClass`; the streak may still animate, but in muted gray instead of bright fire color.
  - Completed cards already get `opacity-60` from `HabitCard.vue`, so `--color-text-muted` must be muted but still readable after wrapper opacity is applied.
- Do not change the habit name, card opacity, card sorting, completion action, revert action, or backend props.
- Do not change the `/streaks/` page. It does not import `HabitCardContent.vue`, so its `🔥` and streak colors should remain as-is automatically.

### `frontend/src/themes/index.js`

Add a `--color-text-muted` CSS variable to every theme's `cssVars` object.

Current theme tokens include `--color-text-primary` and `--color-text-secondary`, but not `--color-text-muted`. Do not use literal Tailwind grays such as `text-gray-400` / `text-gray-500` for this feature, because those escape the theme system and can become unreadable across the existing themes.

Guidance for values:

- Choose a muted text color per theme that is less prominent than `--color-text-secondary`.
- Keep enough contrast for completed Today cards, because `HabitCard.vue` applies `opacity-60` to completed card content.
- For dark themes, avoid values that are so dark they disappear after `opacity-60`.
- For light themes, avoid values that are too close to `--color-text-secondary` if the resulting completed streak still competes with incomplete bright streaks.

### `frontend/src/app.css`

Add `--color-text-muted` to the default `@theme` block and to each `[data-theme="..."]` CSS override.

The values must mirror `frontend/src/themes/index.js` so the token works both before and after JS theme hydration.

Because the frontend uses Tailwind v4, adding `--color-text-muted` to the `@theme` block generates the `text-text-muted` utility class through the same mechanism as the existing `text-text-primary` and `text-text-secondary` classes.

### `frontend/tests/themes.spec.js`

Extend `REQUIRED_CSS_VARS` with `--color-text-muted`.

The theme schema test currently requires only:

- `--color-text-primary`
- `--color-text-secondary`
- accent, danger, background, and streak-fire variables

Adding `--color-text-muted` to `frontend/src/themes/index.js` without updating this test would leave the new token outside the enforced theme contract.

### `frontend/src/components/HabitCard.vue`

No behavior change should be necessary here, but verify the data flow:

- `HabitCard.vue` passes `habit` and `streakClass` into `HabitCardContent.vue`.
- It computes `streakClass` from `getStreakFireClass(props.habit.streak)`.
- Keep passing `streakClass` unchanged.

Keep the conditional presentation inside `HabitCardContent.vue`, because that is where the streak block is rendered and the change is limited to the Today card content.

`streakClass` is an animation utility from `useThemeAnimation.js`, not a color class. Depending on the active theme it can be `streak-fire-pulse-glow`, `streak-fire-flicker-flame`, `streak-fire-bounce`, or empty. Completed Today streaks should keep this animation behavior; only the color changes.

## Pre-implementation Checks

Before implementing the component change, inspect `frontend/src/animations.css` for the three streak animation utilities:

- `streak-fire-pulse-glow`
- `streak-fire-flicker-flame`
- `streak-fire-bounce`

If those rules or their keyframes reference `--color-streak-fire`, hard-coded fire colors, `color`, `text-shadow`, `filter`, `drop-shadow`, or other visual effects that would keep completed streaks looking bright, choose one implementation path before coding:

- Strip `streakClass` for completed habits, if the animation itself is tightly coupled to fire-colored styling.
- Or add a small muted-state CSS class that overrides the conflicting properties without `!important`, then apply it only to completed Today streaks.

If the animation utilities only animate opacity/transform/brightness and do not enforce fire colors, keep applying `streakClass` on completed habits as planned.

## Algorithm

1. In `HabitCardContent.vue`, derive whether the streak block should be muted from `habit.completedToday`.
2. For each streak element:
   - If `habit.completedToday` is false:
     - use the current fire color class and current animation class.
   - If `habit.completedToday` is true:
     - use `text-text-muted` for both the emoji and the text.
     - keep the existing animation class.
3. Use a single conditional class expression for the color class, such as `habit.completedToday ? 'text-text-muted' : 'text-streak-fire'`, so completed streaks remove `text-streak-fire` from the DOM instead of merely overriding it.
   - Do this by moving the current static `text-streak-fire` classes into `:class`.
   - Final markup should keep static classes limited to non-color utilities and bind `[conditionalColorClass, streakClass]`.
4. Account for the existing completed-card wrapper opacity:
   - `HabitCard.vue` applies `opacity-60` when `habit.completedToday` is true.
   - Choose `--color-text-muted` values that remain readable after that opacity is applied.
5. Confirm the streak block remains visible and readable for completed habits, but no longer competes visually with incomplete habit streaks.
