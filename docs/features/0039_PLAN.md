# 0039 - Dark Focus Theme: Remove Undo Toast and Show "No Reward" Popup

## Description

Two Dark Focus theme-specific changes to the habit completion flow:

1. **Remove the undo toast** that appears after completing a habit (the small popup at the bottom with "Undo" button).
2. **Show a "No reward" popup** when the user loses the reward roll. Currently, the `RewardCelebration` overlay only appears when `got_reward` is true. For Dark Focus, it should also appear with a "No reward" message when `got_reward` is false.

## Current Behavior

After completing a habit in `Dashboard.vue` (`completeHabit` function, line 121):
- The backend sets `completionFlash` with `got_reward: true/false` (see `/src/web/views/dashboard.py`, lines 146-160).
- On success, `Dashboard.vue` always calls `showUndo()` (line 153) which displays `UndoToast.vue`.
- The `RewardCelebration` overlay only shows when `flash?.got_reward && flash?.reward_name` (line 145).
- Dark Focus uses `completionCelebration: 'fade-quiet'` and `displayMode: 'expand-from-card'`, so it renders `RewardQuiet.vue` inside `RewardCelebration.vue`.

## Files to Change

### 1. Theme config: `frontend/src/themes/index.js`

Add a new theme-level config key to the `dark_focus` theme definition (line 383):
- `showUndoToast: false` -- signals that the undo toast should be suppressed.
- `showNoRewardPopup: true` -- signals that the reward celebration should also appear on loss.

Add both keys with their default values to the `DEFAULTS` object (line 58) so other themes are unaffected:
- `showUndoToast: true` (default -- all existing themes keep the undo toast)
- `showNoRewardPopup: false` (default -- only Dark Focus opts into "no reward" popup)

### 2. Dashboard: `frontend/src/pages/Dashboard.vue`

In the `completeHabit` function's `onSuccess` callback:

- **Undo toast**: Wrap the `showUndo()` call (line 153) in a conditional that checks `themeConfig.value.showUndoToast !== false`. When the theme disables it, skip calling `showUndo()`.

- **No-reward popup**: After the existing `if (flash?.got_reward && flash?.reward_name)` block (line 145), add an `else if` branch. When `flash?.got_reward === false` AND `themeConfig.value.showNoRewardPopup === true`:
  - Set `celebrationData.rewardName` to a "no reward" label (e.g., "No reward this time").
  - Set `celebrationData.piecesEarned` and `celebrationData.piecesRequired` to `null`.
  - Set `celebrationData.cardRect` from the card element.
  - Set `celebrationVisible.value = true`.

### 3. Reward celebration components

The `RewardQuiet.vue` component (used by Dark Focus via `fade-quiet`) currently hardcodes "Reward earned" as the header text (line 7). It needs to handle the "no reward" case:

- Accept a new optional `isNoReward` prop (Boolean, default false).
- When `isNoReward` is true, change the header from "Reward earned" to "No reward" (or similar) and optionally adjust styling (e.g., muted color instead of accent).

`RewardCelebration.vue` needs to pass the new prop through:
- Accept `isNoReward` prop.
- Pass it to the dynamically rendered `celebrationComponent` (line 36).

`Dashboard.vue` needs to pass `isNoReward` to `RewardCelebration`:
- Add `isNoReward` to `celebrationData` reactive object.
- Set it to `true` when showing the no-reward popup, `false` when showing a real reward.

### 4. Other reward components (optional, for completeness)

`RewardDefault.vue` and `RewardToast.vue` should also accept the `isNoReward` prop and adjust their display text, even though Dark Focus does not use them. This keeps the interface consistent if other themes opt into `showNoRewardPopup` later.

## Summary of Data Flow

1. Backend always returns `completionFlash` with `got_reward: true` or `got_reward: false`.
2. `Dashboard.vue` checks the theme config to decide whether to show the undo toast and/or a no-reward popup.
3. `RewardCelebration.vue` receives a new `isNoReward` prop and forwards it to the inner celebration component.
4. `RewardQuiet.vue` (and siblings) render different text based on `isNoReward`.
