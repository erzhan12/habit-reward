# Feature 0036: Web Interface (Mobile-First Dashboard + Habit Tracking)

## Description

Build a mobile-first, minimalistic web interface using **Django + Inertia.js v1.2 + Vue 3 + Tailwind CSS** with a **dark mode** visual theme. The web UI replaces the existing Streamlit dashboard and adds habit completion functionality. Authentication via **Telegram Login Widget** (existing users only).

**Design principles**: Fast, responsive, not overwhelming, dark-themed, mobile-first.

## Authentication: Telegram Login Widget

### Flow

1. User visits the web app → sees login page with Telegram Login Widget button
2. User taps "Log in with Telegram" → Telegram OAuth popup
3. Telegram returns user data (`id`, `first_name`, `last_name`, `username`, `photo_url`, `auth_date`, `hash`) via JS callback
4. Frontend POSTs this data to Django backend (`POST /auth/telegram/callback/`)
5. Backend verifies HMAC-SHA-256 hash using bot token:
   ```
   secret_key = SHA256(TELEGRAM_BOT_TOKEN)
   data_check_string = sorted "key=value\n" pairs (excluding hash)
   valid = HMAC_SHA256(data_check_string, secret_key) == received_hash
   ```
6. Backend looks up `User` by `telegram_id` — if not found, return error ("Please use the Telegram bot first")
7. If found, create Django session → redirect to dashboard

### Prerequisites

- Run `/setdomain` on @BotFather to register the web domain
- Add `TELEGRAM_BOT_USERNAME` to `.env`

### Files to Create/Modify

- `src/web/views/auth.py` — `telegram_login_page` (GET, renders Login page), `telegram_callback` (POST, verifies hash, creates session)
- `src/web/utils/telegram_auth.py` — `verify_telegram_auth(data: dict, bot_token: str) -> bool` hash verification
- `frontend/src/pages/Login.vue` — Login page with Telegram widget embed

### `main.js` — App Entry Point

```js
import { createApp, h } from "vue"
import { createInertiaApp } from "@inertiajs/vue3"
import Layout from "./components/Layout.vue"
import "./app.css"

createInertiaApp({
  resolve: (name) => {
    const pages = import.meta.glob("./pages/**/*.vue", { eager: true })
    const page = pages[`./pages/${name}.vue`]
    page.default.layout = page.default.layout || Layout
    return page
  },
  setup({ el, App, props, plugin }) {
    createApp({ render: () => h(App, props) })
      .use(plugin)
      .mount(el)
  },
})
```

## Pages & Django Views

### Page 1: Dashboard (Today's Habits) — Main Screen

**Route**: `GET /`
**Django view**: `src/web/views/dashboard.py` → `dashboard(request)`
**Vue page**: `frontend/src/pages/Dashboard.vue`

**Props passed via Inertia**:
```python
inertia.render(request, "Dashboard", props={
    "habits": [
        {
            "id": 1,
            "name": "Running",
            "weight": 10,
            "streak": 5,
            "completed_today": False,
        },
        ...
    ],
    "stats": {
        "completed_today": 3,
        "total_today": 7,
        "total_points_today": 45,
    },
})
```

**Data sources**:
- `habit_service.get_active_habits_pending_for_today(user.id)` — pending habits
- `habit_log_repository.get_logs_for_date(user.id, today)` — today's completions
- `streak_service.get_current_streak(user.id, habit.id)` — per-habit streaks

**UI**: List of `HabitCard` components. Each card shows habit name, weight badge, streak count. Tapping "Done" sends `POST /habits/{id}/complete` via Inertia form, shows instant check animation, then `UndoToast` for 5 seconds.

**Habit completion endpoint**: `POST /habits/<habit_id>/complete/`
- Django view calls `habit_service.process_habit_completion()`
- Returns updated habit list via Inertia redirect back to Dashboard
- Undo: `POST /habits/<habit_id>/revert/` calls `habit_service.revert_habit_completion()`

### Page 2: Streaks & Progress Overview

**Route**: `GET /streaks`
**Django view**: `src/web/views/streaks.py` → `streaks_page(request)`
**Vue page**: `frontend/src/pages/Streaks.vue`

**Props**:
```python
{
    "habits": [
        {
            "id": 1,
            "name": "Running",
            "current_streak": 12,
            "longest_streak": 30,  # from max streak_count in habit_logs
            "weight": 10,
            "completions_this_week": 5,
        },
        ...
    ],
    "summary": {
        "total_completions": 342,
        "active_habits": 7,
        "best_streak": {"habit_name": "Meditation", "count": 45},
    },
}
```

**Data sources**:
- `streak_service.get_current_streak()` per habit
- `HabitLog.objects.filter(user=user).aggregate(Max('streak_count'))` for longest
- `HabitLog.objects.filter(user=user, last_completed_date__gte=week_start).count()` for weekly

**UI**: Sorted list of habits by streak (descending). Each row shows habit name, fire emoji + streak count, mini progress bar for week. Summary card at top with total stats.

### Page 3: Habit History / Calendar View

**Route**: `GET /history`
**Django view**: `src/web/views/history.py` → `history_page(request)`
**Vue page**: `frontend/src/pages/History.vue`

**Props**:
```python
{
    "current_month": "2026-02",
    "completions": {
        # habit_id -> list of dates completed this month
        1: ["2026-02-01", "2026-02-02", "2026-02-05", ...],
        2: ["2026-02-01", "2026-02-03", ...],
    },
    "habits": [
        {"id": 1, "name": "Running"},
        {"id": 2, "name": "Meditation"},
    ],
}
```

**Query params**: `?month=2026-02` (defaults to current month), `?habit=1` (optional filter)

**Data sources**:
- `HabitLog.objects.filter(user=user, last_completed_date__range=(month_start, month_end))` grouped by habit

**UI**: `CalendarGrid` component showing a month view. Each day cell shows colored dots for each completed habit. Tapping a day shows which habits were done. Previous/next month navigation. Habit filter dropdown at top.

### Page 4: Reward Progress Cards

**Route**: `GET /rewards`
**Django view**: `src/web/views/rewards.py` → `rewards_page(request)`
**Vue page**: `frontend/src/pages/Rewards.vue`

**Props**:
```python
{
    "rewards": [
        {
            "id": 1,
            "name": "Coffee",
            "pieces_earned": 3,
            "pieces_required": 5,
            "status": "PENDING",  # PENDING | ACHIEVED | CLAIMED
            "is_recurring": True,
        },
        ...
    ],
    "claimed_rewards": [
        {
            "id": 2,
            "name": "New Book",
            "claimed_at": "2026-02-15",
        },
        ...
    ],
}
```

**Data sources**:
- `reward_repository.get_active_by_user(user.id)` — active rewards
- `reward_progress_repository.get_by_user(user.id)` — progress for each
- `reward_progress_repository.get_claimed_non_recurring_by_user(user.id)` — claimed

**UI**: Grid of `RewardCard` components. Each card shows reward name, progress bar (pieces_earned/pieces_required), status badge. Achieved rewards show a "Claim" button. Claimed section below (collapsible).

**Claim endpoint**: `POST /rewards/<reward_id>/claim/`
- Calls `reward_service.mark_reward_claimed()`
- Inertia redirect back to Rewards page

### Auth-Required Middleware

All web routes except `/auth/*` require an authenticated session. Middleware checks `request.user.is_authenticated` and redirects to `/auth/login/` if not.

Since Inertia uses Django sessions (not JWT), we log the user in with `django.contrib.auth.login()` after verifying the Telegram widget data.
