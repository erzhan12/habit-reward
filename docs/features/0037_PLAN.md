# Feature 0037: Advanced Theme Engine â€” Layout, Interaction & Animation Variants

## Description

Redesign the theme system from scratch. Currently themes only change colors and minor CSS class tokens. The new system makes each theme a completely distinct experience: different layouts (grid vs list, compact vs spacious), different interaction patterns (button vs checkbox vs swipe-to-complete), unique animations (completion celebrations, card entrances, streak fire), theme-specific fonts, and fully custom reward celebrations. All pages get full customization per theme.

Replace the current 7 color-only themes with 8 new themes, each with a strong visual/UX personality. Build architecture-first with 2 themes, then add the remaining 6 incrementally.

## Architecture

### Phase 1: Theme Engine (data layer + infrastructure)

Build the extensible config schema and rendering infrastructure. Validate with 2 themes: **Clean Modern** (simple baseline) and **Gamified Arcade** (most complex â€” swipe, particles, XP bars).

#### 1A. Extended Theme Config Schema

Extend `frontend/src/themes/index.js`. Each theme config gains these new top-level keys:

```
{
  // --- existing keys (kept) ---
  name, icon, description, cssVars, classes, layout, navStyle,

  // --- new keys ---
  font: {
    family: 'Inter, system-ui, sans-serif',   // CSS font-family value
    import: null,                               // Google Fonts URL or null for system
    weight: { normal: '400', medium: '500', bold: '700' },
    size: { base: '1rem', sm: '0.875rem', xs: '0.75rem' },
  },

  interactions: {
    habitComplete: 'button-right' | 'button-left' | 'button-bottom'
                  | 'checkbox' | 'toggle' | 'swipe-reveal',
    // Desktop fallback for swipe: renders as button-right
  },

  animations: {
    cardEntrance: 'none' | 'fade-in' | 'slide-up' | 'stagger-fade',
    completionCelebration: 'none' | 'scale-up' | 'burst-particles'
                          | 'fade-quiet' | 'spring-bounce' | 'warm-glow'
                          | 'matrix-cascade' | 'confetti-emoji',
    hoverMicro: 'none' | 'scale' | 'shadow-lift' | 'glow-border',
    streakFire: 'none' | 'pulse-glow' | 'flicker-flame' | 'bounce',
  },

  reward: {
    displayMode: 'expand-from-card' | 'toast',
    // Component name for fully custom rendering per theme
    celebrationComponent: 'RewardDefault' | 'RewardParticles'
                        | 'RewardMatrix' | 'RewardConfetti' | ...,
  },

  pageLayout: {
    habitList: 'list' | 'grid-2' | 'grid-3',
    density: 'spacious' | 'normal' | 'compact',
    rewardList: 'grid-2' | 'list',
  },
}
```

#### 1B. Font Loading System

**File to create**: `frontend/src/composables/useThemeFont.js`

Algorithm:
1. On theme apply, read `font.import` from config
2. If non-null, dynamically inject `<link>` tag into `<head>` (or use `FontFace` API)
3. Set `--font-family` CSS var on `<html>` and apply via `font-family: var(--font-family)` on `body` in `app.css`
4. Cache loaded fonts to avoid re-fetching on theme switch
5. Show content with system font first, swap when custom font loads (FOUT strategy)

#### 1C. Interaction Resolver

**File to create**: `frontend/src/composables/useThemeInteraction.js`

Reads `themeConfig.interactions.habitComplete` and returns the correct component to render for the Done action. This lets `HabitCard.vue` dynamically switch between:

- `HabitDoneButton.vue` â€” standard button (position varies: left/right/bottom via prop)
- `HabitDoneCheckbox.vue` â€” styled checkbox toggle
- `HabitDoneToggle.vue` â€” iOS-style toggle switch
- `HabitDoneSwipe.vue` â€” swipe-to-reveal (mobile); falls back to `HabitDoneButton.vue` on desktop

Detection of mobile vs desktop: check `window.matchMedia('(pointer: coarse)')` or screen width.

Algorithm for swipe-reveal:
1. Track touch start X position on `touchstart`
2. On `touchmove`, calculate deltaX and translate the card content
3. When deltaX exceeds threshold (e.g. 100px), reveal the green checkmark panel underneath
4. On `touchend`, if past threshold â†’ emit `complete` event, animate card snap-back with checkmark shown
5. If not past threshold â†’ spring back to original position

#### 1D. Animation System

**File to create**: `frontend/src/composables/useThemeAnimation.js`

Provides:
- `cardEntranceClass(index)` â€” returns CSS classes/styles for staggered entrance based on `animations.cardEntrance`
- `completionAnimation(cardEl)` â€” triggers the completion celebration based on `animations.completionCelebration`
- `streakFireClass(streakCount)` â€” returns CSS animation class based on `animations.streakFire`

For particle/confetti effects: use lightweight canvas-based approach (no heavy library). Create `frontend/src/utils/particles.js` with a small canvas overlay that spawns particles from a given DOM element's bounding rect.

Algorithm for expand-from-card reward:
1. On habit completion, get bounding rect of the HabitCard element
2. Create a portal element positioned at the card's center
3. Animate scale from 0 â†’ 1 with the theme's specified animation type
4. Display reward content (name, pieces, progress)
5. Run theme-specific celebration (particles, confetti, matrix text, etc.)
6. Auto-dismiss after 3s or on tap

#### 1E. Theme Transition System

Smooth cross-fade between themes on switch:

Algorithm:
1. Before applying new theme, snapshot current state
2. Apply new theme CSS vars and classes
3. Use `View Transition API` (`document.startViewTransition()`) if supported
4. Fallback: cross-fade overlay with 300ms opacity transition

**File to modify**: `frontend/src/composables/useTheme.js` â€” wrap `applyTheme()` in View Transition API.
