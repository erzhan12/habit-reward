# Feature 0034: My Rewards Filtering & Ordering

## Description

Improve the "My Rewards" view by:
1. **Hide claimed rewards** — don't show rewards with `claimed=True` status
2. **Reorder unclaimed rewards** — sort by percentage of filled pieces (highest first), then Achieved rewards, then never-won rewards (0 pieces) at the bottom
3. **Remove status line** — since claimed rewards are gone and status is implied by the progress bar
4. **Add TODO** for a new "Claimed Rewards" menu button in the Rewards menu (one-time rewards only)

## Sort Order (top to bottom)

1. **Pending rewards** (0 < pieces_earned < pieces_required) — sorted by `pieces_earned / pieces_required` descending (highest % first)
2. **Achieved rewards** (pieces_earned >= pieces_required, not claimed) — ready to claim
3. **Never-won rewards** (pieces_earned == 0) — no progress yet

## Files to Change

### 1. `src/core/repositories.py` — `RewardProgressRepository.get_all_by_user()`
- Currently returns all progress entries ordered by `reward__name`
- Change: Filter out `claimed=True` entries
- Change: Order by computed percentage descending is not possible in ORM easily, so return unclaimed entries and let the service/handler sort in Python

### 2. `src/services/reward_service.py` — `get_user_reward_progress()`
- Currently calls `progress_repo.get_all_by_user(user_id)` and returns the list directly
- Change: Sort the returned list in Python using the ordering algorithm:
  1. Separate into 3 groups: pending (pieces > 0), achieved, never-won (pieces == 0)
  2. Sort pending group by `pieces_earned / pieces_required` descending
  3. Concatenate: pending + achieved + never-won

### 3. `src/bot/formatters.py` — `format_reward_progress_message()`
- Remove the status line (`FORMAT_STATUS`) from the output
- Remove the "Ready to claim!" line for achieved rewards (the ⏳ emoji in the name already signals this)
- Keep: emoji prefix, reward name, progress bar with pieces count

### 4. `TODO.md`
