# 0040 — Auto-Refresh on Day Change

## Problem

When a new day starts, the web page still shows stale data (yesterday's completions, old streaks). The user must manually navigate away and back, or refresh the browser, to see the new day's habits.

## Solution

Create a `useDayChange` composable that detects day boundaries via two triggers:
1. **Midnight timer** — schedule a `setTimeout` for the next midnight, then `router.reload()`
2. **Tab focus** — on `visibilitychange`, compare the current date to a stored "last known date"; if different, `router.reload()`

Wire it into `Layout.vue` so **all pages** auto-refresh silently (Inertia `router.reload()`, no toast or banner).

## Files to Change

### New: `frontend/src/composables/useDayChange.js`

Composable logic:

1. On mount, record `lastKnownDate = today()` (a `YYYY-MM-DD` string from the **browser's local time** — this is good enough since the backend resolves the authoritative "today" from the user's stored timezone on each request).
2. Compute ms until next local midnight: `new Date(year, month, day+1) - Date.now()`. Set a `setTimeout` for that duration.
3. When the timeout fires:
   - Call `router.reload()` to silently refresh page props.
   - Update `lastKnownDate` to the new date.
   - Schedule the next midnight timer (for the following day).
4. Listen to `document.addEventListener('visibilitychange', ...)`:
   - When `document.visibilityState === 'visible'`, check if `today() !== lastKnownDate`.
   - If day changed, call `router.reload()` and update `lastKnownDate`.
5. On unmount, clear the timeout and remove the event listener.

Return nothing (fire-and-forget composable), or optionally expose a reactive `currentDate` ref if useful later.

### Modify: `frontend/src/components/Layout.vue`

- Import and call `useDayChange()` in `<script setup>`.
- No template changes needed.

### Keep as-is: `frontend/src/pages/Dashboard.vue`

- Keep `useRealtimeSync` in Dashboard (the `paused` logic during reward celebrations is page-specific).
- `useDayChange` in Layout handles the day-boundary refresh for all pages including Dashboard.

## Algorithm: Midnight Timer

```
function scheduleMidnightTimer():
  now = new Date()
  tomorrow = new Date(now.getFullYear(), now.getMonth(), now.getDate() + 1)
  msUntilMidnight = tomorrow - now
  // Add 1-second buffer to avoid firing just before midnight due to clock drift
  midnightTimer = setTimeout(onDayChange, msUntilMidnight + 1000)

function onDayChange():
  lastKnownDate = today()
  router.reload()
  scheduleMidnightTimer()   // re-arm for next day
```

## Algorithm: Tab Focus Check

```
function onVisibilityChange():
  if document.visibilityState !== 'visible': return
  if today() === lastKnownDate: return
  lastKnownDate = today()
  router.reload()
  // Also re-schedule midnight timer since we may have missed it
  clearTimeout(midnightTimer)
  scheduleMidnightTimer()
```

## Edge Cases

- **Multiple tabs**: Each tab independently detects day change and reloads — this is fine since `router.reload()` is idempotent.
- **Computer sleep/wake**: The visibility change handler covers this — when the user opens the laptop the next morning, the tab becomes visible and detects the day changed.
- **Clock drift**: The 1-second buffer on the midnight timer prevents firing a fraction of a second before midnight.
- **Login page**: `Layout.vue` is not used on the Login page (it has no layout), so no day-change detection fires there — which is correct.

## Unit Tests

### File: `frontend/src/composables/__tests__/useDayChange.test.js`

Use Vitest with fake timers (`vi.useFakeTimers()`).

1. **Midnight timer fires and reloads**: Set system time to 23:59:58, mount composable, advance timers by ~62s, assert `router.reload()` was called.
2. **Tab focus triggers reload on day change**: Mount composable, change faked date to next day, dispatch `visibilitychange` with `document.visibilityState = 'visible'`, assert `router.reload()` called.
3. **Tab focus does NOT reload same day**: Mount, dispatch `visibilitychange` without changing date, assert `router.reload()` NOT called.
4. **Cleanup on unmount**: Mount and unmount, assert timeout is cleared and event listener removed.
5. **Re-arms timer after midnight**: After first midnight fire, assert a new timer is scheduled.
